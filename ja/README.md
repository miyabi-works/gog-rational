# gog-rational

## Default Deny for AI Agents.

既存のSKILL.mdは「どうやるか」を書いている。
gog-rationalは「やっていいか」を書いている。

---

## なぜこれが必要か

AIエージェントはメールを誤送信し、ファイルを間違った相手に共有し、深夜2時に全社員に通知する。

壊れているからではない。「やってはいけないこと」を誰も教えていないからだ。

既存のSKILL.mdはコマンドの使い方（操作マニュアル）を書いている。
gog-rationalは「何が揃ったら実行していいか」（仕様書）を書いている。

---

## どうやって判断するか

初期状態は常に「実行不可」。
照合条件を全て満たして初めて「実行可能」になる。
**許可されていないことは、全部止まる。**

実行前・実行中・確定時の**3段階**で照合する。
全て適合した場合のみ実行される。
1件でも不適合なら止まって理由を報告する。

削除・共有・全体通知など不可逆な操作は必ず人間に確認を求める。
AIが判断するのではなく、人間が決める。

## カスタマイズ

各SKILL.mdは自由に書き換えられます。
照合条件を緩めることも厳しくすることも可能です。
ただし照合①（開始前チェック）の削除は推奨しません。

照合条件はMarkdownで記述されているため、
**プログラミングの知識がなくても編集できます。**

## 設計思想・研究

本実装は「構造化仕様に基づく階層的適合判定および外部確定処理の制御方法」（特許出願済み・日本）を
AIエージェントのSKILL.mdとして応用したものです。

設計思想の核心：
- **仕様書が資産**：コードではなく仕様書に照合条件を記述する
- **デフォルト禁止**：許可されたことしか実行できない構造
- **説明可能性**：なぜ止まったかをログとして残す
- **人間との協調**：不可逆操作は必ず人間が最終判断する
